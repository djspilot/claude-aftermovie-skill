# aftermovie

GoPro Quik-style aftermovie generator, driven by a prompt. Point Claude (or the CLI) at a folder of clips + a song, get a beat-synced MP4 back.

This is a Claude Code skill. It runs locally via ffmpeg + librosa; no cloud.

## Install

```bash
# clone, then:
brew install ffmpeg                            # if you don't have it
bash scripts/setup.sh                          # creates ~/.skills-data/aftermovie/venv
```

`setup.sh` is idempotent. It installs the package into a local venv so your global Python stays clean and runs `aftermovie doctor` to confirm everything's wired up.

## Use it from Claude Code

The skill lives at `SKILL.md`. Once `~/.claude/skills/aftermovie` points at this repo, ask Claude things like:

> Make me an aftermovie from `~/Movies/Iceland2026` using `~/Music/song.mp3`

Claude will analyze the clips, pick beats, and render an MP4.

## Use it from the CLI

```bash
~/.skills-data/aftermovie/venv/bin/aftermovie auto \
  --clips ~/Movies/Iceland2026 \
  --song ~/Music/song.mp3 \
  --output ~/Movies/iceland.mp4 \
  --theme cinematic
```

Or stage it (much faster on a second iteration since `analyze` is the slow step):

```bash
aftermovie analyze --clips <folder> --out catalog.json
aftermovie score   --catalog catalog.json --song <track> --out plan.json
# inspect plan.json, tweak entries
aftermovie render  --plan plan.json --output out.mp4
```

`aftermovie doctor` reports the state of ffmpeg, deps, LUTs, and optional extras.

## Themes

Four built-in bundles (LUT + music level + speed-ramp policy):

- `cinematic` — neutral cool, lifted blacks, music -10dB
- `punchy` — high-contrast, saturated, music -6dB
- `chill` — desaturated, warm, no speed ramps
- `nostalgic` — film-look fade, yellow highlights

Pass `--theme <name>`. Drop your own `.cube` files into `assets/luts/` to add more.

## Output

H.264 video, AAC audio, 1080p @ 30fps by default. Override with `--res 3840x2160` or `--fps 60`. Vertical (`--aspect 9:16`) and square (`--aspect 1:1`) are supported.

The scorer logs *why* it picked each clip (`hilight_tag`, `high_accel_jump`, `motion_peak`, etc.) into `plan.json`.

## Development

```bash
~/.skills-data/aftermovie/venv/bin/pip install -e ".[dev]"
~/.skills-data/aftermovie/venv/bin/pytest
```

Test fixtures are generated by `python tests/fixtures/build.py` and committed as binaries (~2MB).

## Repository layout

```
aftermovie/
  SKILL.md                        # entrypoint Claude reads
  pyproject.toml                  # package + deps
  src/aftermovie/                 # the implementation
    cli.py                        # argparse dispatcher
    config.py                     # paths, defaults, themes
    types.py                      # ClipInfo, Plan, Candidate
    telemetry/                    # GoPro GPMF + HiLight parsers
    analyze/                      # motion, audio, clip orchestrator
    score/                        # song analysis + scoring
    render/                       # ffmpeg filtergraphs + pipeline
    mcp_server/                   # MCP tools (phase 2)
  assets/luts/                    # color LUTs (.cube)
  references/                     # editing recipe, themes, GPMF format
  tests/                          # pytest suite
  scripts/
    aftermovie.py                 # back-compat shim → aftermovie.cli:main
    setup.sh                      # one-time installer
```
